3. System Features
3.1 Search, Filter and Browse Movies
3.1.1 Description and Priority
In our website, users can search movies by title or by actor/actrees. They can also filter movies
based on movie genres like comedy, action, science fiction, horror, dram etc. Priority = High.
3.1.2 Stimulus/Response Sequences
Stimulus: User enters text in the search bar.
Response: Frontend sends the text to backend, backend queries the database for matching movies or
actors and returns results to be displayed.
Stimulus: User selects one or multiple genres to filter movie results.
Response: System performs a database query to find movies with the selected genres and returns
results.
Stimulus: User applies filters such as release year range, rating threshold, or sorting options.
Response: System retrieves movies that match all selected filters and sorting preferences.
Software Requirements Specification for Movie Database Management Website Page 4
Stimulus: User selects a sorting option for the displayed movies.
Response: Our system will reorder the current movie list based on the selected sorting criteria and
refresh the display without losing applied filters.
Stimulus: User scrolls to the bottom of a movie listing page.
Response: System loads the next page of results using pagination or infinite scrolling.
3.1.3 Functional Requirements
Search.Text: Our system shall allow users to search movies by entering text in a search field
Search.Autocomplete: The system shall provide search suggestions as users type.
Search.Results: The system shall display search results with movie poster, title, year, and rating.
Browse.Categories: The system shall provide browsing options for Popular, Top Rated, Upcoming,
and Now Playing movies.
Filter.Genre: The system shall allow filtering by one or multiple genres.
Filter.Year: The system shall allow filtering by release year range.
Filter.Rating: The system shall allow filtering by minimum rating threshold.
Filter.Clear: The system shall provide a clear all filters button to reset filtering options.
Sort.Options: The system shall allow sorting by rating, popularity, release date, or alphabetical
order.
Sort.Order: The system shall allow toggling between ascending and descending order for all
sorting options.
Pagination: Our system shall implement pagination for browsing results to efficently function.

3.3 User Authentication and Profile
3.3.1 Description and Priority
In our website, users can sign up by creating accounts, login and manage their profiles information.
Priority = High.
3.3.2 Stimulus/Response Sequences
Stimulus: A user creates an account by using their information like username, mail address and a
password.
Response: Our system will generate the hashed password then stores the user information to
database.
Stimulus: A registered user can login to its account by using mail address and passsword.
Response: Our system will generate a JWT token for the logged in user to authorize user for more
functionality.
Stimulus: A logged in user can logout from the account.
Software Requirements Specification for Movie Database Management Website Page 6
Response: Our system will logout the user from the system.
Stimulus: A registered user can send a request to registered mail address to reset the account
password.
Response: Our system will send an e-mail for reseting the user password with a new one which will
be hashed and update the database.
Stimulus: A registered user can manage profile information like username and password etc.
Response: Our system allow users to change profile information such as username or password and
update the database with the new user information.
3.3.3 Functional Requirements
Auth.Register: The system shall allow users to create accounts with email and password.
Auth.Validate.Email: The system shall validate email format and check if the email is already
registered in the database.
Auth.Validate.Password: The system shall enforce password requirements with minimum 8
characters, at least one uppercase letter, one number, and one special character.
Auth.Login: The system shall authenticate users with email and password.
Auth.Token: The system shall generate a secure JWT token upon successful login that expires after
7 days.
Auth.Logout: The system shall allow users to logout and terminate their session.
Auth.Password: The system shall provide password reset functionality via email.
Auth.Failed: The system shall limit failed login attempts to 5 tries within 15 minutes before
temporarily locking the account.
Profile.View: The system shall display user profile with username, join date, and statistics.
Profile.Edit: The system shall allow users to update their profile information and preferences.
Profile.Avatar: The system shall allow users to upload or select a profile avatar.
Profile.Delete: System shall allow users to delete their account.
3.4 Rate and Review Movie
3.4.1 Description and Priority
In our website, registered users can rate a movie and create a review or update an existing review.
Priority = Medium.
3.4.2 Stimulus/Response Sequences
Stimulus: A user can rate a movie based on a 5 star level or 10 points. User can also create a review
for that movie to share his/her feelings, feedback about the movie.
Response: Our system takes the user rate and review object and sends to the backend which we will
save the rate and review information in the database by linking to the user who made the review and
to the movie that is reviewed.
Stimulus: A user can update the rating and the review made to a movie.
Software Requirements Specification for Movie Database Management Website Page 7
Response: Our system can update the previously assigned user rate and review values for a spesific
movie.
Stimulus: A user can delete the rating and the review made to a movie.
Response: Our system can delete the previously assigned user rate and review values for a spesific
movie.
Stimulus: Generate the average rating of the movie.
Response: Our system will update the rating of a movie each time a new review is added or
updated.
Stimulus: Display the ratings and the reviews of the movie made by the registered users.
Response: In our frontend we will show the rating values and the reviews made by the registered
users clearly.
Stimulus: A user marks another user's review as helpful or likes it.
Response: Our system will increment the helpful count for that review and use this metric for
sorting reviews by most helpful.
Stimulus: A user reports a review as inappropriate or spam.
Response: Our system will flag the review for admin moderation and hide it from public view if it
receives multiple reports.
3.4.3 Functional Requirements
Rating.Submit: The system shall allow users to rate movies on a scale of 1-10 or 1-5 stars.
Rating.Update: The system shall allow users to update their previously submitted rating.
Rating.Calculate: The system shall calculate the rating of the movie after anoter review is made or
updated.
Review.Write: The system shall allow users to write text reviews minimum 50 characters,
maximum 1000 characters.
Review.Edit: The system shall allow users to edit their own reviews within 24 hours of posting.
Review.Delete: The system shall allow users to delete their own reviews.
Review.Display: The system shall display user reviews sorted by most recent or most helpful.
Review.Like: System shall allow users to like or upvote helpful reviews.
Review.Report: System shall allow reporting inappropriate reviews.
Review.Helpful: The system shall allow users to mark reviews as helpful or like them.
Review.Report: The system shall allow users to report reviews as inappropriate with a reason
selection.
Review.Spoiler: The system shall allow users to mark their review as containing spoilers with a
warning tag.
Review.Timestamp: The system shall display when each review was posted and if it was edited.

3.5 Manage Watchlist
3.5.1 Description and Priority
In our website, registered users can create and manage personel watchlist. Priority = High.
3.5.2 Stimulus/Response Sequences
Stimulus: A user requests to create a watchlist for a spesific genre or save the movies wanted to
watch later.
Response: If the user is registered we will allow to create watchlist to save movies to it.
Stimulus: A user can add or remove a movie from a watchlist.
Response: Our system will allow registered users to create watchlist to save movies to it.
3.5.3 Functional Requirements
Watchlist.Create: System shall allow creating multiple watchlists.
Watchlist.Add: The system shall allow registered users to add movies to their watchlist.
Watchlist.Remove: The system shall allow users to remove movies from their watchlist.
Watchlist.Rename: System shall allow renaming a watchlist.
Watchlist.Delete: System shall allow deleting a watchlist.
Watchlist.View: The system shall display all movies in a user's watchlist.
Watchlist.Status: The system shall indicate whether a movie is already in the user's watchlist.
Watchlist.Sort: The system shall allow sorting watchlist by date added, title, or release date.
Watchlist.Count: The system shall display the total number of movies in the watchlist.
Watchlist.Movie.Status: The system shall allow users to mark movies as watched without
removing them from the watchlist.
Watchlist.Notes: The system shall allow users to add personal notes to movies in their watchlist.
3.6 Admin Panel and Content Management
3.6.1 Description and Priority
In our website, admin users can manage movie database content, moderate user reviews, and view
system statistics. Admins have special privileges to maintain database quality. Priority = Medium.
3.6.2 Stimulus/Response Sequences
Stimulus: An admin user logs in with admin credentials.
Response: Our system will verify the admin role from the database and grant access to the admin
panel with all management features.
Stimulus: An admin adds a new movie to the database.
Response: Our system will validate the movie information, check for duplicates, and add the new
movie to the database with all provided details.
Software Requirements Specification for Movie Database Management Website Page 9
Stimulus: An admin edits existing movie information.
Response: Our system will update the movie details in the database and log the changes with
timestamp and admin username.
Stimulus: An admin reviews reported user reviews.
Response: Our system will display all reported reviews with the number of reports and reasons,
allowing the admin to approve, remove, or warn the user.
3.6.3 Functional Requirements
Watchlist.Create: System shall allow creating multiple watchlists.
Watchlist.Add: The system shall allow registered users to add movies to their watchlist.
Admin.Login: The system shall authenticate admin users with special admin credentials.
Admin.Dashboard: The system shall provide an admin dashboard with system overview and
statistics.
Admin.Movie.Add: The system shall allow admins to add new movies to the database with
complete details.
Admin.Movie.Edit: The system shall allow admins to edit any movie information in the database.
Admin.Movie.Delete: The system shall allow admins to remove movies from the database with
confirmation.
Admin.Review.Moderate: The system shall allow admins to view, approve, or remove user
reviews.
Admin.User.Manage: The system shall allow admins to view user accounts, suspend accounts, or
ban users.
Admin.Reports: The system shall display all user-reported content for admin review.
Admin.Log: The system shall maintain an activity log of all admin actions with timestamps.
